# Code Review Skill

## 起動条件
- ユーザーがコードレビュー依頼
- PR 作成前
- リリース前

## チェック項目

### 1. 設計
- [ ] Carmack: 計測根拠あるパフォーマンス改善か
- [ ] Martin: 単一責任原則に従っているか
- [ ] Pike: シンプルか / 早期リターン使用か
- [ ] 既存類似機能の重複でないか

### 2. コード品質
- [ ] 関数引数 ≤ 3
- [ ] ネスト深さ ≤ 3
- [ ] `any` 型なし (例外時はコメント)
- [ ] 全関数 docstring あり
- [ ] マジックナンバー定数化

### 3. テスト
- [ ] 単体テスト追加
- [ ] 異常系カバー
- [ ] カバレッジ 80%+ (リスクゾーン 95%+)
- [ ] E2E への影響確認

### 4. セキュリティ
- [ ] 入力バリデーション
- [ ] SQL/XSS インジェクション対策
- [ ] シークレット直書きなし
- [ ] 依存ライブラリの脆弱性スキャン pass

### 5. i18n
- [ ] ハードコード文字列なし
- [ ] `t('key')` 経由
- [ ] key は階層構造

### 6. ドキュメント
- [ ] CHANGELOG.md 更新
- [ ] 関連 docs/*.md 更新
- [ ] API 変更時はバージョニング

## 出力形式
- `[CRITICAL]` 修正必須
- `[MAJOR]` 強く推奨
- `[MINOR]` 改善提案
- `[NIT]` スタイル

各指摘は: ファイル:行 / 問題 / 修正例
