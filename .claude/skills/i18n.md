# i18n Skill

## 起動条件
- 新機能追加 (UI 文字列含む)
- 翻訳追加依頼
- 文字列ハードコード検出

## 設計原則

### キー命名
- 階層構造: `domain.feature.element[.state]`
- 例:
  - `timeline.clip.split` → "クリップ分割"
  - `export.dialog.title` → "エクスポート"
  - `error.network.timeout` → "通信タイムアウト"

### キー追加手順
1. `i18n/keys.ts` に key 定義
2. `i18n/ja.json` (基準言語) に翻訳追加
3. `i18n/en.json` に英訳追加
4. 他言語は段階的追加 (auto-translate + 人手レビュー)

### コンポーネント側
```tsx
// 禁止
<button>分割</button>

// 推奨
<button>{t('timeline.clip.split')}</button>
```

### 補完
- `t('key', { count: 5 })` で複数形対応
- `t('key', { name: 'foo' })` で変数埋め込み
- ICU MessageFormat 準拠

## 1000 言語戦略

### Tier 1 (手動翻訳)
日本語、英語、中国語(簡/繁)、韓国語、スペイン語、フランス語、ドイツ語、ポルトガル語、ロシア語、アラビア語

### Tier 2 (自動翻訳 + レビュー)
上位 100 言語

### Tier 3 (自動翻訳のみ)
残り 900 言語

### フォールバック
1. ユーザー言語
2. 言語ファミリー上位 (例: 関西弁 → 日本語)
3. 英語
4. キー名表示

## 翻訳ワークフロー

### 追加時
- `i18n/sync.ts` で全言語ファイルに key 追加
- 未翻訳は `[TODO]` マーカー
- CI で `[TODO]` 残存をチェック

### 検証
- 全言語で UI 崩れチェック (RTL / 長文)
- 文字幅対応 (CJK / 欧文)
- pluralization テスト

## 禁止事項
- 文字列直接埋め込み
- 連結による文生成 (`t('hello') + ', ' + name` 禁止)
- locale ハードコード
