# Refactor Skill

## 起動条件
- ユーザーがリファクタ依頼
- 重複コード検出時
- ファイルサイズ > 1000行 検出時

## 手順

### 1. 事前準備
- 全テスト pass 確認 (`npm test`)
- 現状の動作仕様を文書化
- カバレッジ計測 (リファクタ前)

### 2. 戦略決定
- **抽出**: 共通化できる箇所を関数/クラスへ
- **分離**: 大きすぎるモジュールを分割
- **統合**: 似た責務の重複を統合
- **改名**: 意図不明な名前を明確に
- **単純化**: 不要な抽象を削除 (Pike原則)

### 3. 実行
- 1関数ずつ変更 + テスト pass 確認
- コミットを小粒度で
- 動作変更ゼロを保証 (テストで検証)

### 4. 検証
- 全テスト pass
- カバレッジ低下なし
- パフォーマンスベンチマーク同等以上
- バンドルサイズ確認

### 5. 完了
- CHANGELOG: `refactor: ...`
- レビュー依頼

## 禁止事項
- リファクタと機能追加の同時実施
- テストなしリファクタ
- 「全部書き直す」アプローチ (incremental のみ)
