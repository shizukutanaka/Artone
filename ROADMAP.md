# Artone v3 — 10年運用ロードマップ

10年間動かせるプロダクトにするための長期計画。30%の品質向上を継続的に達成。

## 設計原則 (10年)

### 不変
- **Web 標準依存のみ**: ベンダーロックイン回避
- **ローカル処理**: データ主権ユーザー側
- **オープンプロトコル**: WebRTC/HLS/RTMP/MIDI/OSC
- **後方互換**: プロジェクトファイルは10年読み込み可能

### 進化
- **段階的アーキテクチャ刷新**: 5年ごとに大規模リファクタ
- **API バージョニング**: v1 → v2 並行運用 → v1 deprecation
- **依存最小化**: 標準化された機能は標準APIへ移行

### 終わり方の設計 (Exit Strategy)

- [ ] 後継メンテナの育成 (バス係数 5+ 確保)
- [ ] アーキテクチャ知識文書化 (新規メンテナが3ヶ月で稼働可能なレベル)
- [ ] 過去10年の設計判断記録 (`docs/ADR/` Architecture Decision Records)
- [ ] サンセット計画 (廃止する場合の利用者通知2年前ルール)
- [ ] 移行ガイド (他 NLE / OTIO 経由) 整備
- [ ] フォーク容易化 (build / test / deploy が完全自動化済み)
- [ ] 商標 / ドメインの引き継ぎ可能化
- [ ] バックアップ戦略 (リポジトリ / ドキュメント / リリース成果物)

### 持続性指標 (Year 10)
- [ ] フルタイム相当メンテナ ≥ 3 名 (年 1500h コミット相当)

## リスク管理

### 技術
- **Web 標準変更**: W3C 仕様追跡 / フィーチャーフラグで切替可能設計
- **ブラウザベンダー方針**: 主要ブラウザ全対応 / フォールバック完備
- **依存ライブラリ消滅**: Vendor in / 自前実装

### 運営
- **メンテナー離脱**: バス係数 3+ / 知識ドキュメント化
- **資金枯渇**: スポンサー / OSS 助成金 / 企業ライセンス
- **法的問題**: ライセンス監査 / GPL コード除外

### セキュリティ
- **ゼロデイ**: 24h 対応体制 / Bug bounty
- **サプライチェーン攻撃**: SBOM 管理 / 署名検証
- **データ漏洩**: ローカル処理徹底 / 暗号化バックアップ
