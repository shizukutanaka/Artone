# recovery/ — データ損失リスクゾーン

## 厳守事項
- 既存リカバリデータを削除するコードは **必ず** トランザクショナル設計
- IndexedDB トランザクションは `readwrite` 終了まで他処理ブロック
- スナップショット書き込みは write-then-rename パターン
- 自動バックアップ間隔の変更は要 user confirm

## 不変性
- `recoverySnapshots` ストアの schema は破壊的変更禁止
- バージョン migration は必ず forward-only
- 過去バージョンのデータは読み込み可能であり続ける

## テスト要求
- カバレッジ 95%+
- クラッシュシミュレーション必須
- 並行書き込み/競合状態テスト必須
- 大容量プロジェクト (10GB+) でのリストア検証

## 変更前確認
1. 既存テスト全 pass
2. データマイグレーション計画
3. ロールバック手順
4. 影響を受けるユーザーへの通知方法
